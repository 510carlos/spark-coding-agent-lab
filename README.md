# Spark Coding Agent Lab

**Reproduce local AI-coding-agent benchmarks on a DGX Spark.**

Clone this, point it at the model you want, and run the *same* agentic bug-fix
task across multiple coding harnesses (Claude Code, OpenCode, Qwen Code, minion)
and multiple local inference engines (DS4 / DeepSeek V4 Flash, vLLM, Ollama).
Out the other end you get a comparable results row: wall time, tokens/sec,
whether the agent self-stopped, and how many tests passed.

> **Honest scope:** this is tuned for a **DGX Spark (GB10, 128 GB unified
> memory, sm_121, CUDA 13, driver ≥580)** — the hardware these stacks were built
> and measured on. It is *not* a "runs anywhere" kit; the engine builds, memory
> limits, and kernel tuning are Spark-specific. That's the point: it's a
> reproducible demo of what one Spark can actually do.

**Goal:** remove the setup overhead. Three host prerequisites plus one command,
instead of a day spent installing toolchains and avoiding out-of-memory lockups.
See [DESIGN.md](DESIGN.md) for the architecture and the reasoning behind it.

## What's inside
```
engines/      one container per inference engine, each exposing an
              OpenAI- and/or Anthropic-compatible HTTP endpoint
  ds4/        DeepSeek V4 Flash 284B (selective Q2 GGUF) — built from source
  vllm-spark/ vLLM on the GB10 sm_121 image (Gemma-26B, etc.)
  ollama/     Ollama (qwen3-coder-next, etc.)
models/       MANIFEST of weights to fetch (NOT baked into images)
bench/        the reusable agent benchmark
  demo-repo/  a tiny Python todo app with one planted bug + 13 tests
  task.md     the verbatim task prompt
  driver.py   headless/tmux driver that runs a harness and auto-handles prompts
harnesses/    wiring for each coding agent (env vars + invocation)
results/      RUN-LOG matrix (regenerated by run.sh)
run.sh        one-command runner: pick an engine + a harness
```

## Requirements
- DGX Spark / GB10 (or sm_121-class), 128 GB unified memory
- NVIDIA driver ≥ 580, CUDA 13, `nvidia-container-toolkit`, Docker
- ~150 GB free disk for model weights
- Node ≥ 20 + `uv` (for the harness clients)

## Quickstart
```bash
git clone <this-repo> && cd spark-coding-agent-lab
# 1. fetch the weights you want (see models/MANIFEST.md)
./models/fetch.sh ds4            # or: ollama pull qwen3-coder-next
# 2. run one engine × one harness on the bug-fix task
./run.sh --engine ds4    --harness claude-code
./run.sh --engine ollama --harness opencode
# 3. read the comparable row appended to results/RUN-LOG.md
```

Run the full harness sweep for one engine with `make matrix ENGINE=ollama MODEL=<name>`.

## ⚠️ One engine at a time
A DeepSeek V4 GGUF (~81 GB) and an 80 B coder (~51 GB) **cannot co-reside** in
128 GB. `run.sh` **stops every other engine before starting the one you asked
for** — this is mandatory on the Spark (running two large models at once will
exhaust unified memory and can hard-lock the host). Don't bypass it.

## The benchmark, in one sentence
Take a repo with a planted `--due` bug (2 of 13 tests failing), hand the agent
the same prompt, and measure: does it fix it (→ 13/13), how fast, and **does it
know when to stop** — the metric token-throughput benchmarks miss.

## Results (from our run)

Full **harness × model** grid — all models warm-loaded, run one at a time.

| Harness | DeepSeek-V4 284B | qwen3-coder-80B | qwen3.6-35B | Ornith-1.0-35B |
|---|---|---|---|---|
| **OpenCode**    | 231 s · 13/13 | **20 s** · 13/13 | 43 s · 13/13 | 43 s · 13/13 |
| **Claude Code** | 172 s · 13/13 | 60 s · 13/13 | 43 s · 13/13 | ✗ template error |
| **Qwen Code**   | 181 s · 13/13 | 31 s · 13/13 | **24 s** · 13/13 | 43 s · 13/13 |

**11 of 12 cells fixed the bug (13/13)** — the model isn't the bottleneck. The lone
failure is a chat-template incompatibility (Ornith via Claude Code's Anthropic
endpoint), not a coding miss. The 284B runs ~170–230 s in *every* row vs 20–60 s for
the small coders — the **model owns wall-time, not the harness**. (minion was dropped
from the grid — it does not reliably self-terminate and ran to the timeout every
time; the script remains in `harnesses/` for reference, with a caveat.)
